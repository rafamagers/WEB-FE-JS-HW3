Understanding JavaScript Async Concepts

1. JavaScript Async

JavaScript is single-threaded, meaning it can only perform one operation at a time. Async operations allow JavaScript to perform non-blocking tasks.

// Example of async operation
console.log("Start");
setTimeout(() => console.log("Async operation"), 1000);
console.log("End");

2. Callback

A callback is a function passed as an argument to another function. It is executed after the first function completes its operation.

// Example of a callback function
function greet(name, callback) {
    console.log("Hello, " + name);
    callback();
}

function sayGoodbye() {
    console.log("Goodbye!");
}

greet("Alice", sayGoodbye);

3. Promise

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It has three states: pending, fulfilled, and rejected.

// Example of a Promise
let promise = new Promise((resolve, reject) => {
    let success = true;
    if(success) {
        resolve("Operation successful!");
    } else {
        reject("Operation failed!");
    }
});

promise.then(result => console.log(result))
       .catch(error => console.log(error));

4. Async/Await

Async/Await is syntactic sugar built on top of promises. It allows asynchronous code to be written in a synchronous manner.

// Example of Async/Await
async function fetchData() {
    try {
        let data = await fetch("https://api.example.com/data");
        console.log("Data received:", data);
    } catch (error) {
        console.log("Error:", error);
    }
}

fetchData();

5. Event Loop

The event loop is responsible for handling asynchronous operations in JavaScript. It continuously checks the call stack and callback queue.

// Example of Event Loop
console.log("Start");

setTimeout(() => {
    console.log("Callback");
}, 0);

console.log("End");
